# Plan 0: Foundation & Workspace

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a buildable Xcode workspace with all four targets (main app, clipboard daemon, Quick Look extension, Shared Swift package) signed with the same Developer ID, sharing an App Group container, with linting and CI in place. No feature code; just the scaffold every later plan builds on.

**Architecture:** Single Xcode workspace at the repo root containing one Xcode project (`MacAllYouNeed.xcodeproj`) with three application/extension targets and one in-repo Swift package (`Shared`) consumed by all three. The Shared package exposes three library modules: `Core`, `UI`, `Platform`. App Group entitlement (`group.com.macallyouneed.shared`) wires the three runtime processes together.

**Tech Stack:** Xcode 15+, Swift 5.9+, Swift Package Manager (in-repo), SwiftLint, SwiftFormat, GitHub Actions, XCTest.

**Reads from spec:** §4 (Bundle layout & module structure), §10 (Code signing & notarization).

**Produces working software:** all four targets build green and launch as empty shells; SwiftLint and SwiftFormat run cleanly; CI passes on a PR.

- [ ] **Step 3: Implement `AppGroup`**

`Shared/Sources/Core/AppGroup.swift`:

```swift
import Foundation

public enum AppGroup {
    public static let identifier = "group.com.macallyouneed.shared"

    /// Test-only override. Set in unit tests to redirect storage to a temp dir.
    public static var containerURLOverride: URL?

    /// Returns the real App Group container URL if the current process has the
    /// entitlement. Runtime targets must use this for signed entitlement checks.
    public static func entitledContainerURL() -> URL? {
        FileManager.default.containerURL(forSecurityApplicationGroupIdentifier: identifier)
    }

    /// Returns whether `containerURL()` is using the unit-test fallback path.
    public static func isUsingFallbackContainer() -> Bool {
        containerURLOverride == nil && entitledContainerURL() == nil
    }

    /// Returns the App Group container URL. Falls back to a per-user temp dir
    /// only when called from a context without the entitlement (e.g. unit tests),
    /// so callers can be exercised without entitlements. Runtime manual checks
    /// must fail if `isUsingFallbackContainer()` is true.
    public static func containerURL() -> URL {
        if let override = containerURLOverride { return override }
        if let url = entitledContainerURL() {
            return url
        }
        let fallback = FileManager.default.temporaryDirectory
            .appendingPathComponent("MacAllYouNeed-\(NSUserName())", isDirectory: true)
        try? FileManager.default.createDirectory(at: fallback, withIntermediateDirectories: true)
        return fallback
    }
}
```

```bash
#!/usr/bin/env bash
set -euo pipefail

WORKSPACE="MacAllYouNeed.xcworkspace"
SCHEME="MacAllYouNeed"

echo "==> SwiftLint"
swiftlint --strict

echo "==> SwiftFormat lint"
swiftformat --lint .

echo "==> Swift package tests"
(cd Shared && swift test)

echo "==> Xcode build"
xcodebuild \
  -workspace "$WORKSPACE" \
  -scheme "$SCHEME" \
  -configuration Debug \
  -destination "platform=macOS" \
  CODE_SIGNING_ALLOWED=NO \
  build

echo "==> Xcode tests"
xcodebuild \
  -workspace "$WORKSPACE" \
  -scheme "$SCHEME" \
  -configuration Debug \
  -destination "platform=macOS" \
  CODE_SIGNING_ALLOWED=NO \
  test
```

- [ ] **Step 2: Make executable and run locally**

```bash

./scripts/ci-build.sh
```

Expected: every section reports success.

## Task 0.9: Add GitHub Actions CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Write workflow**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4

      - name: Select Xcode 15
        run: sudo xcode-select -s /Applications/Xcode_15.4.app/Contents/Developer

      - name: Install SwiftLint and SwiftFormat
        run: brew install swiftlint swiftformat

      - name: Run CI script
        run: ./scripts/ci-build.sh
```

- [ ] **Step 2: Commit and push**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add GitHub Actions workflow"
git push origin <branch>
```

- [ ] **Step 3: Verify CI passes**

Open the PR (or push to main if working solo). Watch the Actions tab. Expected: green build.

If `macos-14` runner is unavailable, fall back to `macos-latest` and pin Xcode via `sudo xcode-select` to the version actually installed on that runner.

## Self-review checklist for the engineer (do this before declaring Plan 0 done)

Run all of:

```bash
xcodebuild -workspace MacAllYouNeed.xcworkspace -scheme MacAllYouNeed -configuration Debug build
xcodebuild -workspace MacAllYouNeed.xcworkspace -scheme MacAllYouNeed -configuration Debug test
(cd Shared && swift test)
swiftlint --strict
swiftformat --lint .
./scripts/ci-build.sh
```

Then verify by hand:

- Launch main app from Xcode. Menu-bar icon appears, no dock icon, popover shows the App Group container path.
- Run ClipboardDaemon scheme. NSLog shows the same App Group container path.
- Press Space on a folder in Finder. Quick Look shows the placeholder text including the same App Group container path.
- None of the runtime targets shows `FALLBACK - entitlement missing`. Fallback is acceptable only in package/unit tests that run without signing entitlements.

If any of these fail, the daemon and the extension are likely missing the App Group entitlement or not signed with the same Developer ID — fix before continuing to Plan 1.
